# 北京星界力科技有限公司官网技术实现文档

## 1. 文档信息

- 文档版本：V1.1
- 目标：明确官网分阶段技术实现路径，确保首版快速上线与后续扩展兼容
- 适用范围：官网项目（前端 Next.js、后端 NestJS、Monorepo 架构）

## 2. 技术目标与原则

- 以业务上线为优先：第一版优先完成高质量静态官网
- 以扩展能力为约束：首版结构需兼容第二版后端与管理端接入
- 以稳定和SEO为底线：保证访问性能、搜索引擎可抓取性、可持续迭代
- 以统一工程管理为核心：采用 Monorepo 管理多端与共享模块

## 3. 总体架构设计（Monorepo）

### 3.1 架构选型

- 前端：Next.js（App Router）
- 后端：NestJS（第二版接入）
- 仓库形态：Monorepo
- 包管理与任务编排：采用统一 workspace 方案（如 pnpm workspace + Turborepo）

### 3.2 目录规划建议

- `apps/web`：官网前端（V1主要交付）
- `apps/api`：NestJS 服务端（V2启动）
- `apps/admin`：后台管理端（V2启动）
- `packages/ui`：跨端共享UI组件
- `packages/config`：ESLint、TypeScript、构建配置共享
- `packages/types`：共享类型定义（V2起主要使用）
- `docs`：需求、背景、技术方案等文档

### 3.3 分阶段边界

- V1：仅实现 `apps/web`，静态内容发布
- V2：引入 `apps/api` 与 `apps/admin`，逐步替换静态内容为动态管理

## 4. V1 交付范围（前端静态官网）

### 4.1 页面范围

- 首页
- 产品体系页
- 教育解决方案页
- 关于我们页
- 联系我们页

### 4.2 内容来源

- 以《需求文档》与《公司背景文档》为唯一内容源
- 企业主体信息严格以营业执照为准
- 品牌叙事、产品命名、对外口径保持一致

### 4.3 多端适配要求

- 断点策略：Mobile / Tablet / Desktop 三端适配
- 导航策略：移动端折叠菜单，桌面端完整导航
- 版式策略：模块化栅格布局，核心信息优先展示
- 交互策略：按钮触达区域、字体层级、可读性优先

## 5. SEO 与可发现性方案

### 5.1 首版 SEO 目标

- 页面可被主流搜索引擎稳定抓取
- 品牌词与核心业务词具备基础排名能力
- 核心落地页具备独立可索引价值

### 5.2 执行要点

- 使用服务端渲染或静态预渲染输出完整 HTML
- 每个页面设置唯一 `title`、`description` 与语义化标题结构
- 输出 `sitemap.xml` 与 `robots.txt`
- 设置规范化链接（canonical）避免重复页面权重分散
- 使用结构化数据（Organization、WebSite 等）增强搜索理解
- 保障首屏性能与 Core Web Vitals 指标

### 5.3 内容侧 SEO 协同

- 页面文案围绕“能力 + 场景 + 结果”组织
- 关键词布局避免堆砌，优先自然语义表达
- 品牌词、产品词、解决方案词按页面分工承载

## 6. Cloudflare 部署策略（V1）

### 6.1 发布范围

- 第一版仅部署前端官网到 Cloudflare
- 后端与管理端暂不部署生产环境

### 6.2 部署形态建议

- 优先使用 Cloudflare Pages 进行持续部署
- 以 Git 分支驱动预览环境与正式环境
- 生产域名启用 HTTPS 与缓存策略
- V1 采用静态导出模式（`next.config.ts` 中 `output: "export"`）
- 构建产物目录为 `apps/web/out`，可直接作为静态站点发布目录

### 6.3 与后续 V2 的衔接

- 预留 API 网关与域名路由策略（如 `api.xxx.com`）
- 保持前端接口层解耦，V2 可平滑接入 NestJS
- 不改变已上线页面 URL 结构，保护SEO积累

## 7. 技术可行性评估

### 7.1 可行结论

- Next.js 静态官网 + Cloudflare 部署方案可快速落地，交付风险低
- Monorepo 能满足后续多应用扩展与共享资产治理
- 当前阶段不引入后端可显著降低复杂度与上线周期

### 7.2 关键风险与应对

- 风险：首版静态内容更新效率低  
  应对：V2 建立后台管理与内容发布流程
- 风险：后续 API 接入导致前端改造范围扩大  
  应对：V1 即采用接口抽象层与模块化目录
- 风险：SEO 依赖内容质量与持续更新  
  应对：建立月度内容迭代机制与页面质量复盘
